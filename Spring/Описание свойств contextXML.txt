1. <bean> - сущность.
 1.1. id/name - имя по которому можем однозначно определить бин. (обязательный)
 1.2. class - путь по которому бин будет искать наш класс. (обязательный)
 1.3. scope - отвечает за то как спринг будет создавать бин. Варианты: prototype или singletone (всегда новый бин или один и тот же бин).
 1.4. lazy-initialization mode - если этот параметр истина, то бин будет создан только тогда когда к нему будет обращение.
 1.5. initialization method - при создании бина будет вызван этот метод. **
 1.6. destruction method - при уничтожении бина будет вызван этот метод. **

 1.7. <constructor-arg> - отвечает за конструктор конкретного класса.
      Если у класса есть не дефолтный конструктор, то указываем этот конструктор и передаем нужные параметры.
  1.7.1. ref - указывается ссылочное свойство конструктора.
  1.7.2. value -указывается примитивное сваойство конструктора.
  1.7.3. index - порядковый нмоер свойства в конструкторе.
  1.7.4. type - тип свойства в конструкторе.

 1.8. <properties> - Внедрение зависимостей с помощью сеттеров. *
  1.8.1. name - указывается название сеттера без префикса set.
  1.8.2. ref - указывается ссылка на бин.
  1.8.3. value - указывается значение.

  *  С помощью "p:" можно(не обязательно) сделать запись короче. См. пример.
  ** initialization method и destruction method прописываются в классе как обычный метод,
     а потом в эти параметры нужно указать имена этих методов. Называться эти методы могут как угодно.
